Deep Agents — LangChain's opinionated agent harness

Batteries-included agent in two lines. Middleware stack with planning, filesystem, sub-agents, summarization, and Anthropic prompt caching built in.

Language Python ≥3.11 License MIT Default model Claude Sonnet 4.6 ↗ langchain-ai/deepagents

TL;DR

Middleware stack, not a monolith. Every capability — planning, filesystem, sub-agents, summarization, memory, skills — is a middleware. You compose your agent by choosing middleware.
Built on LangGraph primitives. Streaming, checkpointing, persistence, and the Command pattern for atomic state updates come free.
Batteries included. create_deep_agent() with zero args returns a working agent that plans, reads/writes files, spawns sub-agents, and auto-compacts.

Analogy: Pi gives you Lego bricks. Deep Agents gives you the Death Star kit. Both can build the same thing — Deep Agents just shipped most of it pre-assembled.

1 · Two-line quickstart

from deepagents import create_deep_agent
agent = create_deep_agent()
result = agent.invoke({
    "messages": [{"role": "user", "content": "Research X and save a report"}]
})

That agent already has: planning (write_todos), filesystem (read_file, write_file, edit_file, ls, glob, grep), shell (execute), sub-agents (task), auto-summarization, Anthropic prompt caching, and AGENTS.md memory loading.

5 · Context Engineering — 5 Coordinated Techniques

🧱 Stacked system prompt

Each middleware appends a content block. Stable prefix blocks get prompt-cached on Anthropic (~80% cost drop on long sessions).

📦 Auto-summarization

At 85% of max_input_tokens: offload old messages to /conversation_history/, replace with LLM-generated summary, keep last 10%.

✂️ Tool-arg truncation

Older write_file / edit_file args get truncated to "first 20 chars + …".

📁 Large result eviction

Results > 20k tokens get evicted to /large_tool_results/; inline is replaced by head/tail preview + file ref.

🔎 Progressive skill disclosure

Only skill name + description + path in system prompt. Full SKILL.md loaded on-demand.

6 · Sub-agents

The task tool spawns ephemeral, isolated sub-agents — a key context-pollution defense.

  • Context isolation — sub-agent only sees its task description
  • Parallel — multiple task calls in one assistant turn run concurrently
  • Inherits middleware (TodoList, Filesystem, Summarization, PromptCaching, PatchToolCalls)
  • Custom personas — set per-agent model, tools, system_prompt

8 · Testing & Evaluation

9 · Deep Dive: Harness × Context × Memory

Deep Agents is the harness whose internals are the LangGraph state machine. Here's how the three layers cooperate.

9.1 · Harness engineering — middleware stack wrapping a LangGraph StateGraph

Each middleware can hook three points: before_agent (once at start), wrap_model_call (every LLM call), and wrap_tool_call (every tool execution). The agent loop itself is a LangGraph StateGraph with two nodes — model and tool — that ping-pong until no tool calls remain.

  • Middleware stack — 10 default middlewares (TodoList → Memory* → Skills* → Filesystem → SubAgent → Summarization → AnthropicCaching → PatchToolCalls → user* → HumanInTheLoop*); each registers tools and wraps calls
  • Command pattern — tools return Command(update={...}) for atomic state updates (messages, files, todos, custom state all mutate transactionally)
  • Recursion limit 1000 — allows deep agent loops; LangGraph would otherwise abort at 25
  • Persistence via checkpointer — pass a Checkpointer (e.g. MemorySaver) and the whole state is persistable to disk / Postgres / Redis
  • Streaming free — LangGraph streams events; the wrapping middleware composes around them transparently

9.2 · Context engineering — 5 cooperating techniques

No single trick fixes context. Deep Agents stacks five: cached system prompt blocks, auto-summarization, tool-arg truncation, large-result eviction, and progressive skill disclosure.

  • Anthropic content blocks — each middleware appends a content block to SystemMessage.content_blocks; cache-eligible prefix stays warm
  • Defensive summarization — first try the normal call; if ContextOverflowError, fall back to summarization (two-phase pattern)
  • Sub-agent context isolation — task tool spawns ephemeral sub-agents with empty history; the noisy work doesn't pollute the main thread
  • Provider-agnostic core — caching is Anthropic-specific but all other techniques work on any tool-calling model
  • compact_conversation tool — agent can manually trigger compaction when switching tasks (proactive context hygiene)

9.3 · Memory management — backend-routed virtual filesystem

Memory in Deep Agents is just files. The BackendProtocol abstracts where files live — LangGraph state (ephemeral), local disk, LangGraph BaseStore (cross-session), composite (routed by path), or a remote sandbox.

  • AGENTS.md is system-prompt memory — MemoryMiddleware loads files into <agent_memory> tags at startup; the agent can edit_file to update them
  • StateBackend is the default — fully ephemeral; pair with a checkpointer if you want it to survive across invocations
  • StoreBackend = persistent memory — backed by LangGraph's BaseStore (in-memory, Postgres, etc.); great for cross-session knowledge
  • CompositeBackend — route specific paths to specific backends, e.g. /memories/ to Store while /tmp/ stays in ephemeral state
  • Trust-the-LLM security — there's no in-prompt safety wall; security lives at the backend / sandbox layer (Runloop, Daytona, Modal)

10 · Strengths & Tradeoffs

💪

Strengths

  • Best-in-class context management
  • Composable via middleware
  • LangGraph persistence + streaming + checkpointing
  • Real eval harness (harbor)
  • Drop-in agent in 2 lines
⚖️

Tradeoffs

  • LangChain/LangGraph dependency surface
  • Middleware ordering matters and isn't obvious
  • Python-first
  • Heavier abstraction than Pi
