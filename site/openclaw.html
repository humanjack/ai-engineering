OpenClaw — A multi-channel productization of Pi

A local-first personal AI assistant daemon. Embeds Pi as its agent runtime; layers on 22+ messaging channels, 122 plugins, 56 skills, cron, hooks, MCP, and disciplined prompt caching.

Language TypeScript (ESM) License MIT Embeds @earendil-works/pi-coding-agent v0.74 ↗ openclaw/openclaw

TL;DR

OpenClaw is Pi, productized. The agent loop is literally Pi's — OpenClaw imports createAgentSession from @earendil-works/pi-coding-agent and embeds it.
One Gateway per host. A long-lived local daemon with a WebSocket control plane. Many clients (CLI, menu bar, mobile, web UI) connect to it.
Everything Pi doesn't ship — cron, hooks, webhooks, MCP server + client, Docker/SSH sandboxes, file-based memory, channel routing — is added as plugins under extensions/.

Analogy: If Pi is the V8 engine, OpenClaw is the entire car — body, dashboard, infotainment, and routes to 22 highway systems.

5 · Prompt Cache Discipline

OpenClaw treats prompt caching as a first-class engineering invariant. From AGENTS.md:

"Prompt cache: deterministic ordering for maps/sets/registries/plugin lists/files/network results before model/tool payloads. Preserve old transcript bytes when possible."

This shows up as:

  • Custom transcript entry type openclaw.cache-ttl marking TTL anchors inside JSONL sessions
  • isCacheTtlEligibleProvider() deciding per-provider whether to emit TTL markers (Anthropic family, Kilocode-routed Anthropic, Google/Gemini)
  • Cache-TTL-aware pruning in pi-hooks/context-pruning.ts
  • Pre-compaction reminder: "save important notes to memory before compacting"

6 · Automations — 4 Surfaces

🪝 Internal hooks

Gateway-level events: command:new/reset/stop, session:compact:before/after, message:received/sent, agent:bootstrap, gateway:startup/shutdown

🧩 Plugin hooks

Loop-level: before_model_resolve, before_prompt_build, before_agent_start, before_tool_call / after_tool_call, before_compaction / after_compaction

⏰ Cron

jobs.json persistence, per-job auth profiles, isolated agent turns, heartbeat-driven wakeups, webhook delivery

📥 Webhooks + Gmail

Inbound HTTP triggers (extensions/webhooks/) + Gmail PubSub watcher

7 · Memory — Bounded Files First, Vectors Optional

OpenClaw's memory is human-readable markdown by default:

  • MEMORY.md — persistent top-level notes
  • memory/YYYY-MM-DD.md — auto-rotated daily entries
  • memory-lancedb plugin — optional vector search atop the markdown
  • memory-wiki plugin — Obsidian-compatible wiki link resolution

Stance: explicit curation > implicit infinite recall.

8 · Strengths & Tradeoffs

💪

Strengths

  • 22+ messaging channels from one daemon
  • Disciplined prompt-cache engineering
  • Plugin SDK with enforced boundaries
  • Pi embedding gives full event-stream control
  • Two well-defined hook levels
⚖️

Tradeoffs

  • Tight coupling to Pi's SDK surface
  • Two hook layers is a learning curve
  • Code plugin vs bundle plugin distinction
  • Large surface area (8,948 production TS files)
